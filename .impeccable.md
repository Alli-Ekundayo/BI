## Design Context

### Users
Primary users are operations leads and founders using this as an additional feature inside a broader BI platform with existing static business-insight dashboards. Their core jobs are to quickly turn natural-language questions into clear data outputs and rapidly visualize results through tables, charts, and plots.

### Brand Personality
Confident, accurate, minimal. The interface should feel trustworthy and efficient, reducing friction for decision-making without visual noise.

### Aesthetic Direction
Visual direction should align with Amazon PartyRock-style approach, while avoiding blue and purple hues. Support both light and dark themes. Since this is shipping soon with an MVP polish bar, prioritize consistency, clarity, and interaction completeness over deep visual experimentation.

### Design Principles
1. Prioritize immediate comprehension: every screen should make the next action obvious.
2. Keep visual language minimal and disciplined: strong hierarchy, low ornamentation, consistent spacing.
3. Communicate trust through accuracy cues: explicit states, clear labels, and predictable behavior.
4. Make visualization-first workflows fast: table/chart/plot outputs should be one-step reachable.
5. Enforce inclusive defaults: WCAG AA contrast, visible focus states, and reduced-motion support.
